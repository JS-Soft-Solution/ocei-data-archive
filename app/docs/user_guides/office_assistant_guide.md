- ‚úì Address is complete and valid

## Decision Making

### Option 1: APPROVE ‚úÖ

**When to approve:**
- All information appears accurate
- Attachments are clear and complete
- No obvious discrepancies
- Application meets minimum standards

**How to approve:**
1. Click **"Approve"** button
2. Confirm action
3. Application moves to **"Submitted to Secretary"**
4. Your name is logged as verifier

### Option 2: REJECT ‚ùå

**When to reject:**
- Missing information
- Unclear or poor quality attachments
- Obvious data entry errors
- Inconsistencies between fields
- Insufficient documentation

**How to reject:**
1. Click **"Reject"** button
2. **Enter clear rejection reason** (required)
3. Be specific about what needs fixing
4. Confirm action
5. Application returns to operator with status **"Rejected by OA"**

---

## Writing Good Rejection Reasons

### ‚ùå Bad Examples:
- "Not good"
- "Fix this"
- "Incomplete"

### ‚úÖ Good Examples:
- "NID copy is blurry and unreadable. Please upload a clearer image."
- "Father's name is missing. This is a required field."
- "Old certificate number format is incorrect. Should be format: ELC-YYYY-NNNN"
- "Attachment for education certificate is missing. Please upload degree certificate."

**Tip:** Imagine you're the operator - what specific information would help you fix the issue?

## Bulk Operations

### Bulk Approve

**Use when:** Multiple applications are clearly valid

1. Check boxes next to applications to approve
2. Click **"Bulk Approve"**
3. System processes all at once
4. Shows success/failure summary

**Caution:** Only use bulk approve when you're certain about quality

---

### Bulk Reject

**Use when:** Multiple applications have the same issue

1. Check boxes next to applications to reject
2. Click **"Bulk Reject"**
3. **Enter a common rejection reason**
4. Confirm action
5. All selected applications get the same reason

**Example common reasons:**
- "Batch contains poor quality scans. Please re-scan attachments clearly."
- "Missing NID copies. Please attach NID for all applications."

## Best Practices

### DO:
- ‚úÖ Review carefully but efficiently
- ‚úÖ Provide constructive rejection feedback
- ‚úÖ Check attachments download correctly
- ‚úÖ Process applications in chronological order (oldest first)
- ‚úÖ Use bulk operations for efficiency
- ‚úÖ Verify NID format (10 or 13 digits)
- ‚úÖ Check certificate number uniqueness

### DON'T:
- ‚ùå Approve without viewing attachments
- ‚ùå Give vague rejection reasons
- ‚ùå Sit on applications for too long
- ‚ùå Approve questionable data "just to move it along"
- ‚ùå Reject for trivial formatting issues

## After You Approve

1. Application status changes to **"Submitted to Secretary"**
2. Secretary reviews for final approval
3. Your verification is logged in audit trail
4. Application leaves your pending queue

---

## Handling Re-submissions

If an operator resubmits after your rejection:
- It appears in your pending queue again
- You can see previous rejection reason in history
- Check if they actually fixed the issues
- If not fixed properly, reject again with more detail
